Five orchestration options

Same emerald family, one distinct shade each. The "pick when" line is the whole decision in a sentence.

Step Functions

AWS Step Functions

A visual state machine that chains tasks with built-in retry, catch, branching, and wait. Serverless service integrations call Glue and Athena directly.

Pick when you want a few AWS services run in order with first-class retries and zero servers to manage. This lab's choice.
Cost: basically free at this scale (per state transition).
Glue Workflows

AWS Glue Workflows

Chains crawlers and Glue jobs with triggers, all inside Glue. No extra service to learn if your whole pipeline already lives in Glue.

Pick when every step is a Glue crawler or Glue job and you never leave Glue.
Cost: no charge for the workflow itself; you pay only for the Glue runs.
MWAA / Airflow

Amazon MWAA

Managed Apache Airflow. Python DAGs, a huge operator ecosystem, rich backfills and dependencies for complex, cross-system pipelines.

Pick when you have many interdependent pipelines and a team that already knows Airflow.
Cost: not free — an environment bills hourly even when idle. Do not provision it for this lab.
EventBridge

EventBridge schedule

A cron or rate timer that fires a target. It is the trigger, not the workflow: it starts something else, like a state machine, on a clock.

Pick when you need to run a pipeline on a daily or hourly schedule. You add this on top.
Cost: basically free (a handful of scheduled invocations a day).
Lambda trigger

Lambda trigger

A short function that reacts to an event, such as a new file landing in S3, and kicks off work. Event-driven glue between services.

Pick when a pipeline should start from an event, not a clock, or you need a tiny bit of custom logic.
Cost: basically free within the generous free tier.
